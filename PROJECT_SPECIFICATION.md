# Kids Task Tracker - Техническое Задание (v1.0)

## 1. Общее Описание

**Название проекта:** Kids Task Tracker  
**Тип приложения:** Кроссплатформенное мобильное приложение (iOS, Android, Web, Desktop)  
**Целевая аудитория:** Семьи с детьми от 7 лет  
**Технологический стек:** 
- Frontend: Flutter + Riverpod
- Backend: Firebase (Auth, Firestore, Cloud Functions)
- Архитектура: Clean Architecture

---

## 2. Роли Пользователей

### 2.1. Родитель (Parent)
- Полный доступ к управлению приложением
- Создание и настройка заданий
- Управление подарками/наградами
- Просмотр статистики детей
- Настройка уведомлений
- Подтверждение выполнения заданий (на определенных этапах)

### 2.2. Ребенок (Child)
- Просмотр назначенных заданий
- Перемещение заданий по этапам Kanban-доски (с ограничениями)
- Просмотр своих наград и баланса
- Получение уведомлений

### 2.3. Отношения между пользователями
- У одного родителя может быть **до 10 детей**
- Ребенок может быть привязан к **нескольким родителям** (родственники)
- Привязка ребенка происходит через систему запросов (child requests)

## 3. Основные Функциональные Требования

### 3.1. Система Авторизации и Регистрации
- Регистрация родителей (email + пароль)
- Регистрация детей (только по приглашению родителя или через код привязки)
- Вход по email/паролю
- Разделение интерфейса по ролям
- Восстановление пароля

### 3.2. Личный Кабинет (ЛК)

#### Для ребенка:
- Просмотр накопленных подарков
- Баланс бонусов/денег
- История выполненных заданий
- Текущие активные задачи

#### Для родителя:
- Список всех детей
- Статистика по каждому ребенку
- Управление подарками
- Настройки приложения
- История операций

### 3.3. Система Заданий (Task Tracker)

#### Типы заданий:
1. **Одноразовые** - выполняются один раз
2. **Периодические**:
   - Ежедневные (например, "сделать уроки")
   - Еженедельные (например, "прибраться в комнате")
   - Ежемесячные
   - Свой кастомный период

#### Поля задания:
- Название
- Описание
- "Цена" (в баллах/деньгах/подарках)
- Периодичность
- Привязка к конкретной Kanban-доске
- Ограничения на перемещение по этапам
- Срок выполнения (опционально)
- Обязательность (для системы штрафов)

#### Предустановленные задания:
- "Помыть посуду"
- "Сделать уроки"
- "Прибрать в комнате"
- И другие базовые шаблоны

### 3.4. Kanban-доски

#### Возможности:
- Несколько досок для разных категорий:
  - Учеба и домашние дела
  - Спортивные достижения
  - Творческие задачи
  - Другие категории

#### Этапы (настраиваемые родителем):
- Гибкое количество этапов
- Названия этапов настраиваются
- Для каждого задания можно настроить:
  - На какие этапы ребенок может перемещать самостоятельно
  - Какие этапы требуют подтверждения родителем
  - На каких этапах отправляются уведомления

#### Пример стандартного workflow:
1. **To Do** (Запланировано)
2. **In Progress** (В процессе)
3. **Review** (На проверке) ← требует подтверждения родителем
4. **Done** (Выполнено) ← только родитель может перевести сюда

### 3.5. Система Наград и Подарков

#### Типы наград:
1. **Баллы/Бонусы** - виртуальная валюта
2. **Подарки-карточки**:
   - Создаются родителем
   - Красивое оформление (иконка, название, описание)
   - Примеры: "Конфетка", "Поход в кино", "Дополнительный час игр"
3. **API-подарки** (Premium):
   - Интеграция со сторонними сервисами
   - Автоматическое выполнение действий
   - Пример: заказ пиццы, активация подписки

#### Механика получения:
- Автоматическое начисление при переводе задания в статус "Выполнено"
- Возможность ручного добавления родителем
- История всех начислений

#### Хранение:
- Вкладка "Подарки/Призы" в ЛК
- Каталог доступных подарков
- История полученных подарков

### 3.6. Система Уведомлений

#### Триггеры:
- Перемещение задания на определенный этап
- Наступление срока выполнения
- Выполнение периодического задания
- Новые подарки доступны
- Штрафы за невыполнение

#### Настройки:
- Родитель настраивает текст уведомления
- Выбор получателя (родитель, ребенок, оба)
- Каналы доставки (push, email, in-app)

### 3.7. Роботы-автоматизации (IFTTT-подобные)

#### Возможности:
- Родитель программирует правила
- Формат: **IF** (триггер) **THEN** (действие)

#### Примеры:
- IF задание "Уроки" выполнено → THEN создать новое задание на завтра
- IF 5 заданий выполнено за неделю → THEN начислить бонус
- IF сегодня понедельник → THEN создать задание "Уроки"

#### Расширенные возможности (Premium):
- Интеграция с внешними API
- Умные цепочки действий
- Кросс-борд автоматизации

### 3.8. Групповые Задачи

#### Особенности:
- Одна задача на нескольких детей
- Общее вознаграждение делится между участниками
- Синхронное или асинхронное выполнение
- Видимость только для участников группы

### 3.9. Система Штрафов

#### Механика:
- За невыполненные обязательные задачи
- Штрафы в баллах/деньгах
- Автоматическое списание по истечении срока
- Уведомления о предстоящем штрафе

## 4. Технические Требования

### 4.1. Архитектура Приложения

```
lib/
├── features/
│   ├── auth/              # Авторизация
│   ├── tasks/             # Управление задачами
│   ├── kanban/            # Kanban-доски
│   ├── rewards/           # Награды и подарки
│   ├── notifications/     # Уведомления
│   ├── robots/            # Автоматизация
│   ├── profile/           # Личный кабинет
│   └── settings/          # Настройки
├── core/                  # Общие компоненты
├── data/                  # Слои данных
├── domain/                # Бизнес-логика
└── presentation/          # UI
```

### 4.2. Структура Базы Данных (Firestore)

#### Коллекции:
- `users` - пользователи (родители и дети)
- `childRequests` - запросы на привязку детей
- `tasks` - задания
- `boards` - Kanban-доски
- `columns` - этапы досок
- `rewards` - подарки
- `userRewards` - полученные награды
- `notifications` - уведомления
- `robots` - правила автоматизации
- `taskLogs` - история перемещений задач

### 4.3. Безопасность

#### Правила Firestore:
- Дети видят только свои задачи
- Родители видят задачи своих детей
- Изоляция данных между семьями
- Валидация на уровне базы данных

#### Аутентификация:
- Firebase Authentication
- Ролевая модель
- Защита облачных функций

### 4.4. Производительность

- Оптимизация количества чтений/записей
- Кэширование на клиенте
- Пагинация больших списков
- Фоновая синхронизация

- Продвинутые роботы-автоматизации
- Расширенная аналитика
- Приоритетная поддержка
